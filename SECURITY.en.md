# Security Policy

## Supported Versions

| Version | Support        |
| :----- | :------------- |
| latest | Active         |

## Reporting a Vulnerability

If you discover a vulnerability in this project, please report it using one of the following methods:

1. **GitHub Issues** — for public discussions (only for non-sensitive issues).
2. **Private message** — via the repository owner's profile, for confidential reports.

Please do not publish vulnerability details in public issues until it is fixed.

## What to Include in the Report

- Description of the vulnerability and potential impact.
- Steps to reproduce.
- Affected code version (commit, tag, or release).
- Possible workarounds (if any).

## Response Time

We aim to respond within 7 business days and provide a fix plan.
